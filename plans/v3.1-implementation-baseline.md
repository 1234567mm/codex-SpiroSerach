# SpiroSearch V3.1 Implementation Baseline

Date: 2026-07-05

## Objective

V3.1 turns the literature/RAG layer into an auditable material-screening factory. RAG finds evidence, but role gates, material ontology, evidence quality, failure modes, and curation state decide whether a material can be ranked as a normal n-i-p Spiro replacement.

## Implemented MVP Boundary

- PDF, supplementary files, object-store payloads, manual inbox files, and full-text chunk exports are excluded from Git.
- Literature intake deduplicates records by DOI, OpenAlex ID, Semantic Scholar ID, or title fingerprint.
- Closed or unavailable full text creates a manual acquisition task instead of attempting unauthorized download.
- Material screening separates material identity from use instance, property observations, and device evidence.
- p-i-n/SAM/interface/barrier evidence is routed to architecture opportunities, not direct n-i-p ranking.
- High-PCE evidence without Spiro comparator, replicates, or stability protocol is penalized and sent to curation.
- Unavailable or synthesis-risk candidates route to `source_or_synthesize`.

## Not Yet Implemented

- Real PostgreSQL/pgvector migrations.
- Real external API clients for OpenAlex, Crossref, Semantic Scholar, Unpaywall, arXiv, or PubMed.
- PDF/SI parsers, OCR, chunking, embedding, rerank, and UI dashboard.
- Human user accounts, permissions, or reviewed-claim promotion workflow.
