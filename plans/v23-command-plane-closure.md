# V23 Controlled Review And Recompute Command Plane Closure

Date: 2026-07-16

## Scope

V23 added a controlled command plane for review decisions and recompute
requests without turning existing read-only surfaces into write surfaces.

Delivered:

- command preflight guardrails for manifest-native, non-legacy runs;
- `ActionRequest` and `ActionResult` contracts plus JSON Schemas;
- authorization, idempotency replay/conflict handling, and optimistic
  run/hash/version preconditions;
- a separate V23 command MCP registry and command tool constructor;
- append-only command action/audit artifacts and recompute job status artifacts;
- static viewer command-state rendering from manifest-discovered artifacts;
- end-to-end replay/security coverage for accepted, duplicate, stale,
  unauthorized, conflict, and read-only registry separation paths.

## Remaining V24/V25 Risks

- V24 admission should consume command outputs as audited facts, not raw command
  payloads or viewer state.
- V24/V25 recompute execution still needs a real dispatcher/status lifecycle;
  V23 only records deterministic command/job state artifacts.
- Any future write-capable transport must preserve read-only registry
  separation and the V23 idempotency/precondition semantics.
