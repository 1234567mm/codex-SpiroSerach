## 1. Boundary

This runbook covers **local workstation** use only.

V28 hosting boundary:

- no hosted deployment
- no cloud credentials requirement for core local workflow
- no external write command plane
- DevContainer/Docker packaging is out of scope for this document because the
  repository currently has no `.devcontainer/` or root `Dockerfile` at the freeze
  SHA

If a step would require hosted services, stop and park it for V29.

## 3. Install

Editable install via uv (preferred):

```powershell
uv run --with-editable . python -c "import spirosearch; print('ok')"
```

If using an existing local venv:

```powershell
.\.venv\Scripts\python.exe -c "import sys; print(sys.version)"
$env:PYTHONPATH = 'src'
```

Notes:

- Default runtime should not require optional scientific binaries (ORCA/xtb) for
  unit tests.
- `uv.lock` may be generated by local `uv` usage; treat it as generated local
  state unless a release process commits it intentionally.
- Do not put secrets into the repo. OpenAlex/LLM keys are optional experimental
  provider settings only.

## 6. Artifact View (local)

Static viewer code lives at:

- `frontend/artifact-viewer/index.html`
- supporting JS/CSS in the same directory

Local viewing pattern:

1. Generate or choose a run directory that contains `run-manifest.json`.
2. Open the static viewer from a local file server or browser workflow used by
   the operator.
3. Discover artifacts from the manifest metadata; do not hard-code filenames in
   new tooling.

Read-only API inventory is defined in `src/spirosearch/readonly_api.py` and is
manifest-driven. Read surfaces must not trigger live provider calls, scoring
mutation, or experiment writes.

## 7. Backup Scope

Align with V25 recovery scope (`src/spirosearch/v25_recovery_runbook.py`):

- `run-manifest.json`
- artifacts referenced by the manifest
- schemas
- command outputs
- handoff artifacts

Also back up, when relevant to a local rehearsal:

- `data/` committed baselines you depend on
- generated `outputs/` run directories you may need to restore
- this runbook and related V28 evidence docs

Backup rule:

- preserve hashes and schema refs
- `external_credentials_required` must remain false for core restore

## 8. Restore Procedure

1. Restore the run directory (or backup bundle) to a clean local path.
2. Confirm `run-manifest.json` is present.
3. For each restored artifact of interest, verify:
   - path resolves
   - `sha256` matches expected value when recorded
   - `schema_ref` is present and schema validates when applicable
4. Use repository readers rather than ad-hoc file edits:

```powershell
$env:PYTHONPATH = 'src'
uv run python - <<'PY'
from pathlib import Path
from spirosearch.artifact_repository import JsonArtifactRepository
repo = JsonArtifactRepository(Path('outputs/screening'))
print(repo.manifest_path if hasattr(repo, 'manifest_path') else 'repo-ready')
print(repo.read_json.__name__)
PY
```

5. Build or consult a V25-style recovery runbook payload for release fixtures:
   - `build_v25_recovery_runbook(...)` fails closed on hash/schema/migration issues.
6. If migration policy is blocked, do not claim restore success.

## 9. Failure Recovery

| Failure | Local response |
| --- | --- |
| Full test gate red | do not claim local readiness; capture failing module names |
| Missing `run-manifest.json` | treat run as unreadable; regenerate or restore from backup |
| Hash mismatch on restore | mark recovery `blocked`; re-copy from known-good backup |
| Missing schema_ref | mark recovery `blocked`; do not silently accept |
| Optional extra import errors (`bo`/`ml`) | keep default runtime green; record extra as unavailable |
| Provider key missing (OpenAlex/LLM) | leave provider experimental/unavailable; do not add secrets to git |
| Custom HTL pilot blocked | expected at V28 P0; do not fabricate molecules |
| Accidental `uv.lock` generation | classify as local generated state; remove if not part of intentional release |
